HTMX question

Is it possible to have multiple trigger types in HTMX? For example, suppose that I want to trigger the request when a select element changes or load and then display the response in an area of my page with id="result". So, if I have the following select element, how would I use/code the htmx attributes for this scenario? Notice that the POST request is already coded.

Megan's answer

Yes, if I wanted to trigger a result appearing when someone selects an item from the dropdown menu OR after 2 seconds pass I would use a comma between those two scenarios, using hx-trigger="click, keyup changed delay:2000ms"

The HTMX attributes that would be worth using in this scenario are:

1. hx-get="/get-image" to populate an image of the selected city

2. hx-target specifies where to put the response

3. hx-trigger="change" - The request happens when the dropdown value changes

1. hx-indicator="#loading" - Shows a loading indicator while the request is in progress
